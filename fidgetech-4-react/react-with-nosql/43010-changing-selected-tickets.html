ğŸ““ 4.3.0.10 Changing Selected Tickets

In the last lesson, we got our list of tickets showing again. However, we can't click on them. In this lesson, we will restore that functionality. In the process, we'll learn about a React Redux Firebase piece of functionality called withFirestore(). We'll also learn how to use a method called get() to retrieve data from Firestore.

If we take a look at our handleChangingSelectedTicket() method in TicketControl.js, it should be obvious why it no longer works:

src/components/TicketControl.js
handleChangingSelectedTicket = (id) => {
  const selectedTicket = this.props.masterTicketList[id];
  this.setState({selectedTicket: selectedTicket});
}

We are no longer storing our tickets in the masterTicketList state slice in our Redux store. In fact, we don't need that state slice at all anymore (and it can be removed from mapStateToProps as well).

Let's update this method to get the specific ticket we need from Firestore.

First, we'll need to import Firestore functionality via React Redux Firebase:

src/components/TicketControl.js
import { withFirestore } from 'react-redux-firebase'

withFirestore() is a wrapper method much like the React Redux connect() method. To use it, we need to wrap this method around the TicketControl component in the final line of code in TicketControl.js:

src/components/TicketControl.js
export default withFirestore(TicketControl);

Just as we do with the connect() method from React Redux, we're using a higher order component to give our component the functionality it needs (the ability to use Firestore).

This makes Firestore available to our application via this.props.firestore.

Now we're ready to update our handleChangingSelectedTicket() method:

src/components/TicketControl.js
handleChangingSelectedTicket = (id) => {
  this.props.firestore.get({collection: 'tickets', doc: id}).then((ticket) => {
    const firestoreTicket = {
      names: ticket.get("names"),
      location: ticket.get("location"),
      issue: ticket.get("issue"),
      id: ticket.id
    }
    this.setState({selectedTicket: firestoreTicket });
  });
}

We can use Firestore's get() method to manually retrieve a collection or a subset of a collection. In this case, we want a ticket with a specific id so we do the following:

this.props.firestore.get({collection: 'tickets', doc: id})

Note that we pass id into a property called doc.

This returns a pending promise. As we know from Intermediate JavaScript, we can chain then() to a promise. But what exactly does our promise return? It doesn't return a ticket. It returns a DocumentSnapshot. A DocumentSnapshot is a Firestore object that contains read-only data of a specified document. The id of the document is readily available but we have to use a get() method to grab each specific property. Note that this is a different get() method than the one we use to actually retrieve the document from Firestore.

That's why we have to do the following to grab a property such as location:

location: ticket.get("location")

We reconstruct a ticket and then set that object to the value of firestoreTicket. Now we have a value to pass into selectedTicket. When we run our application and click on a ticket, we'll be able to go to the ticket detail page (though the edit and delete functionality won't be working).

A few things to note here. This isn't necessarily the most efficient way to get the value of the selectedTicket. Why make a request to the Firestore database when the value of our tickets is already in our application? Well, it's a perfect opportunity to see how get() works - and Firestore is fast enough - and our application small enough - that it has minimal impact on our application's performance.

Also, what would happen if an id isn't correctly passed into this function - or if the id no longer exists in the database? This could definitely be an issue in a larger application. A best practice here would be to add error handling and perhaps a try/catch block. If you'd like to refresh yourself on these concepts from Intermediate JavaScript, you may want to try adding this functionality on your own.

In this lesson, we learned about React Redux Firebase's withFirestore() function, which we can use to add functionality to a class component. withFirestore() adds Firestore to a component's props, allowing us to use it for any kind of requests, not just get(). We could use it to update or delete tickets, for instance. Over the next few lessons, we'll get update and delete functionality working in our application again.
