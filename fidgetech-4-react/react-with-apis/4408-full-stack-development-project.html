📓 4.4.0.8 Full Stack Development Project

In this course section, you will have the opportunity to practice your skills as a full stack developer. During your time at Epicodus, you have learned both front end and back end development skills. Full stack developers are expected to work on both the front and back end. Not surprisingly, they tend to get paid more than developers that specialize in just front or back end.

While we've worked on both front and back end skills throughout our time at Epicodus, now we'll have the chance to practice something entirely new - combining what we've learned from our back end class (C#/.NET) with what we've learned of JavaScript and React.

You'll have the opportunity to do a longer project in 4.4.1, where you will combine a back end API that you or another student made with a React front end. Alternately, you may choose to combine forces with another student and do a mini group project that combines a back end API with React. Note that these projects will not be presented formally - however, they are a great opportunity to improve your skills and put together a portfolio piece.

For the 4.4.1 project, you will do the following:

  • First, find an API that either you or a classmates built during your back end language class. If other students built this project, make sure to clone or fork the project. If you have questions about how to go about this, please ask an instructor. Ultimately, it's up to you to decide how you want to proceed with this project. While a React front end is required, you can spend as much or as little time as you want on building out the back end API further.

By the end of the course section, your project should include the following:

  • Two repositories, one for the back end and one for the front end
  • The README for the API should document all endpoints, include local server deployment instructions, and include a link to the repository for the front end project
  • The README for the front end should include local server deployment instructions as well as a link to the repository for the API
  • The React front end should receive data from the back end API and the UI should update based on this data

Here are some other goals to consider:

  • Deployment of both the API and the front end
  • Authentication/authorization so that only the front end React application can communicate with the API (challenging - but an opportunity for students that are particularly interested in working on their back end language further)

While teachers won't be evaluating this project, don't pass up this opportunity to work on a portfolio piece and to build your skills as a potential full stack developer.
