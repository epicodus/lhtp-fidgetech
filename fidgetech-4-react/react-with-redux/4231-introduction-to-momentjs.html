📓 4.2.3.1 Introduction to Moment.js

Note: You are not expected to incorporate any of the concepts from the lessons on the Moment.js library for this section's project.

Over the next several lessons, we will add a key piece of functionality to our help queue. Users will be able to see how long a ticket has been open just like with the Epicodus help queue.

In this lesson, we'll learn about Moment.js, a JavaScript library used for managing time and dates.

Then, in the next lesson, we'll learn about a React component's lifecycle - and how to write lifecycle methods.

Finally, we'll put all the pieces together and add this functionality to our help queue.

Moment.js​

Moment.js is a popular JavaScript library for working with time. We can use Moment.js to manipulate and parse time - which is exactly what we'll use it for.

We'll start by adding it to our project:

npm install moment@2.24.0

The documentation for Moment.js is extensive. We recommend checking it out - there are many use cases where it can add valuable functionality to an application.

We will import it into components where it is needed with the following import statement:

import Moment from 'moment';

Then we can create an instance of a moment by calling new Moment().

Once we have an instance of a moment, we can take advantage of all kinds of functionality, whether that is comparing two times, formatting time, or something else.

In our case, we are only interested in seeing how much time has elapsed since a ticket was added to the queue. Moment.js offers the fromNow() method to parse the amount of time that has passed since a specific moment. You can check out the documentation on this method here.

If a ticket was put in seven minutes ago, the output from Moment.js will be 7 minutes ago. We can remove the suffix ago by calling fromNow(true). Passing the true boolean into this method means that the output would be 7 minutes instead of 7 minutes ago.

For our particular use case, this is all we need to know about Moment.js for our help queue. We need to be able to create a moment when a ticket is instantiated and then we need to check to see how much time has passed since the ticket was put in.

Now that we know a little about Moment.js, we're ready to take a closer look at the lifecycle of React components - as well as lifecycle methods we can use to automatically update a React component. We'll do that in the next lesson.
